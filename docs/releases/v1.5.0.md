# MygramDB v1.5.0 Release Notes

**Release Date:** 2026-03-23
**Type:** Feature Release
**Previous Version:** v1.4.0

---

## Overview

Version 1.5.0 introduces the `verify_text` post-filter for eliminating n-gram false positives, a Docker-based benchmark environment with 1.1M Wikipedia articles, and a namespace refactor from `mygramdb::utils` to `mygram::utils`.

**Highlights:**

- **verify_text Post-Filter** — Eliminates n-gram false positives by verifying matches against original text; configurable per-table or globally
- **Docker Benchmark Environment** — One-command benchmark setup (`make bench-up`) with 1.1M Wikipedia dataset, comparing MygramDB vs MySQL FULLTEXT
- **Search Pipeline Extraction** — Refactored search execution into a composable pipeline for better testability and extensibility
- **Atomic File Writer** — Safe snapshot writes with atomic rename to prevent corruption on crash
- **Namespace Cleanup** — Renamed `mygramdb::utils` to `mygram::utils` across the codebase for consistency

---

## New Features

### 1. verify_text Post-Filter

**Type:** Feature

N-gram indexes inherently produce false positives when bi-gram substrings match unrelated content. The `verify_text` option adds a post-filter step that checks each candidate result against the original document text, eliminating false matches.

- Configurable globally (`memory.verify_text`) or per-table
- Modes: `off` (default for backward compatibility), `on` (verify all results)
- Benchmark shows 10–2800% false positive reduction depending on query

**Files Changed:**

- `src/server/search_pipeline.cpp`, `src/server/search_pipeline.h`
- `src/config/config.cpp`, `src/config/config.h`

### 2. Docker Benchmark Environment

**Type:** Feature

A self-contained benchmark environment that downloads and loads 1.1M Wikipedia articles (EN 1M + JA 100K) into MySQL, then compares MygramDB's in-memory n-gram index against MySQL FULLTEXT with ngram parser.

- `make bench-up` — Downloads 225MB seed data, loads into MySQL, starts MygramDB
- `make bench-run` — Runs latency, count, consistency, and concurrent throughput benchmarks
- `make bench-down` — Stops and cleans up
- Version-aware entrypoint: automatically enables `verify_text` on v1.5+
- Uses `ghcr.io/libraz/mygram-db:latest` — no local build required

**Files Changed:**

- `docker-compose.bench.yml`
- `support/docker/mysql/bench-init/02-load-seed.sh`
- `support/docker/bench-entrypoint.sh`
- `support/seed/benchmark.py`
- `Makefile`

### 3. Search Pipeline Extraction

**Type:** Refactor

Extracted search execution logic from monolithic handler into a composable `SearchPipeline` class, improving testability and enabling features like `verify_text` to be cleanly inserted as pipeline stages.

**Files Changed:**

- `src/server/search_pipeline.cpp`, `src/server/search_pipeline.h`

### 4. Atomic File Writer

**Type:** Feature

Snapshot files are now written atomically using write-to-temp + rename, preventing partial/corrupted snapshots if the process crashes during a write.

**Files Changed:**

- `src/storage/atomic_file_writer.cpp`, `src/storage/atomic_file_writer.h`

## Changed

### Namespace Rename: mygramdb::utils → mygram::utils

**Type:** Refactor
**Severity:** Low (internal only, no public API change)

Renamed the internal utility namespace from `mygramdb::utils` to `mygram::utils` for consistency with the project naming convention. This change affects all source and test files but has no impact on the public API, configuration, or wire protocol.

## Migration Guide

No breaking changes to configuration or wire protocol. Upgrading from v1.4.0 is a drop-in replacement.

To enable the new false-positive filter, add to your configuration:

```yaml
memory:
  verify_text: all
```

Or set the environment variable:

```bash
MEMORY_VERIFY_TEXT=on
```
