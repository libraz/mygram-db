# MygramDB v1.5.1 Release Notes

**Release Date:** 2026-04-01
**Type:** Bug Fix / Packaging
**Previous Version:** v1.5.0

---

## Overview

Version 1.5.1 expands platform support with Debian/Ubuntu packages and EL10 RPM packages, fixes several build and CI issues, and hardens the release pipeline.

**Highlights:**

- **DEB package support** — Ubuntu 22.04 (Jammy) and 24.04 (Noble) packages with systemd integration
- **EL10 RPM support** — RHEL/AlmaLinux/Rocky Linux 10 packages alongside existing EL9
- **Package verification test suite** — Automated install, startup, health check, and search validation across all target distros
- **Multi-distro CI release pipeline** — GitHub Releases now publish 8 packages (2 RPM versions x 2 DEB versions x 2 architectures)
- **SIGILL fix** — Portable build flag prevents illegal instruction crashes on non-build CPUs

---

## New Features

### 1. Debian Package (.deb) Support

**Type:** Feature

Added full Debian packaging infrastructure for Ubuntu 22.04 LTS (Jammy) and 24.04 LTS (Noble). Packages include systemd service, automatic user creation, and proper directory layout.

- `support/deb/Dockerfile.debbuild` — Parametric build container (`UBUNTU_VERSION` arg)
- `support/deb/debian/` — Standard Debian packaging files (control, rules, postinst, service)
- `support/deb/test-deb-build.sh` — Local build verification script

### 2. EL10 RPM Support

**Type:** Feature

Extended RPM packaging from EL9-only to EL9 + EL10. The RPM build Dockerfile is now parametric via `EL_VERSION` build argument, using AlmaLinux as the base image.

- `support/rpm/Dockerfile.rpmbuild` — Parametric with `ARG EL_VERSION=9`
- `support/rpm/test-rpm-build.sh` — Added `--el-version` option

### 3. Package Verification Test Suite

**Type:** Feature

Comprehensive test script (`support/testing/test-pkg-verify.sh`) that validates packages across all target distros in Docker:

- **Basic tests (12 checks):** Binary existence, `--version`, `--help`, `-t` config validation, library dependencies, file layout, systemd service, user creation
- **Integration tests (20 checks):** MySQL startup, server startup, health endpoints (`/health/live`, `/health/ready`, `/health/detail`), CLI search (English + Japanese), CLI INFO command

### 4. Multi-Distro Release Pipeline

**Type:** Feature

Expanded `release.yml` from 2 jobs (EL9 x86_64/aarch64) to 8 jobs across 4 distros x 2 architectures. Release artifacts are filtered to main packages only (no debuginfo/debugsource/src).

## Bug Fixes

### 1. SIGILL on Non-Build CPUs

**Type:** Bug Fix
**Severity:** High

Added `MYGRAMDB_PORTABLE_BUILD=ON` to Docker build configuration. Without this flag, the compiler could emit instructions (e.g., AVX-512) not supported by the target CPU, causing illegal instruction crashes.

**Files Changed:**

- `docker-compose.yml`, `docker-compose.bench.yml`

### 2. Coverage Target Shell Escaping

**Type:** Bug Fix
**Severity:** Low

Added `VERBATIM` to coverage CMake custom targets for correct shell argument escaping on all platforms.

### 3. Sanitizer CI Configuration

**Type:** Bug Fix
**Severity:** Low

Switched sanitizer workflows to manual-only triggers and fixed coverage `label-exclude` pipe escaping.

### 4. Bench Compose Normalization

**Type:** Bug Fix
**Severity:** Low

Enabled `MEMORY_NORMALIZE_LOWER` in benchmark Docker Compose configuration for consistent search behavior.

## Changed

- Upgraded third-party dependencies and hardened CI pipeline security
- Excluded LOAD tests from default CI runs; labeled query parser performance tests as SLOW
- Applied clang-format (Google style) across source and test files
- MySQL GPG key handling improved with `--nogpgcheck` for EL10 compatibility

## Migration Guide

No migration steps required. This is a packaging and build infrastructure release with no application-level breaking changes.
