# MygramDB v1.5.2 Release Notes

**Release Date:** 2026-04-09
**Type:** Feature / Bug Fix
**Previous Version:** v1.5.1

---

## Overview

Version 1.5.2 adds MySQL 9.x compatibility, including support for the new VECTOR column type (type code 242) introduced in MySQL 9.0. It also fixes MySQL 8.4+ authentication issues and modernizes Docker configurations for cross-version compatibility.

**Highlights:**

- **MySQL 9.x compatibility** — Binlog parser handles VECTOR type columns without errors
- **VECTOR type support** — `MYSQL_TYPE_VECTOR` (242) parsed as 4-byte-prefixed blob, matching LONG_BLOB encoding
- **Authentication fix** — `MYSQL_OPT_GET_SERVER_PUBLIC_KEY` enables `caching_sha2_password` without SSL
- **Version-switchable e2e tests** — Test against MySQL 8.4 and 9.6 with `--mysql-version` flag
- **Docker 9.x compatibility** — Removed deprecated `--binlog-format` and `--mysql-native-password` options

---

## MySQL 8.4 vs 9.6 Source Compatibility

A full source-level comparison of MySQL 8.4.8 and 9.6.0 was conducted across all binlog/replication areas. Key findings:

| Area | Change | Impact |
|------|--------|--------|
| Event type numbers (33–42) | Identical | None |
| MYSQL_RPL struct / binlog API | Identical | None |
| GTID / Tagged GTID wire format | Identical | None |
| Transaction Payload (type 40) | Identical | None |
| DECIMAL encoding | Identical | None |
| Row event flags | New bit 4 (`USE_SQL_FOREIGN_KEY_F`) | Safe — unknown bits ignored |
| Table_map optional metadata | New TLV field 12 (`VECTOR_DIMENSIONALITY`) | Safe — unknown fields skipped |
| Column types | New `MYSQL_TYPE_VECTOR = 242` | **Required handling** |

**Conclusion:** The only breaking change for binlog consumers is the new VECTOR column type. All other wire formats are byte-for-byte identical.

## New Features

### 1. MYSQL_TYPE_VECTOR Support (MySQL 9.0+)

**Type:** Feature

Added support for `MYSQL_TYPE_VECTOR` (type code 242), introduced in MySQL 9.0 for storing fixed-dimension float vectors. VECTOR columns use the same wire encoding as BLOB (metadata byte for length prefix size + data).

**Changes:**

- `src/mysql/table_metadata.h` — Added `VECTOR = 242` to `ColumnType` enum
- `src/mysql/binlog_util.h` — Added `case 242` to `calc_field_size()` (fallthrough to BLOB logic)
- `src/mysql/binlog_event_parser.cpp` — Added `case ColumnType::VECTOR` to TABLE_MAP metadata parsing
- `src/mysql/rows_parser.cpp` — Added VECTOR field decoding (hex string output)

**Backward compatibility:** No impact on MySQL 8.x replication. Type 242 never appears in 8.x binlog streams.

### 2. MySQL Version-Switchable E2E Tests

**Type:** Feature

E2E tests can now run against different MySQL versions via environment variable or CLI flag.

```bash
# Default (MySQL 8.4)
./e2e/run-all.sh

# MySQL 9.6
./e2e/run-all.sh --mysql-version 9.6

# VECTOR-specific tests only
./e2e/run-all.sh --mysql-version 9.6 -m mysql9
```

**Changes:**

- `e2e/docker/docker-compose.yml` — `image: mysql:${MYSQL_VERSION:-8.4}`
- `e2e/run-all.sh` — Added `--mysql-version` option parsing
- `e2e/conftest.py` — Added `mysql_version` and `setup_vector_table` fixtures

### 3. VECTOR Replication E2E Tests

**Type:** Feature

Four new e2e tests verify that tables with VECTOR columns replicate correctly. Tests are automatically skipped on MySQL 8.x.

- `test_insert_with_vector_column` — INSERT with VECTOR data
- `test_update_with_vector_column` — UPDATE VECTOR column
- `test_delete_with_vector_column` — DELETE rows with VECTOR
- `test_batch_inserts_with_vector` — Batch INSERT with VECTOR

## Bug Fixes

### 1. MySQL 8.4+ Authentication (caching_sha2_password)

**Severity:** High — Connection failure on MySQL 8.4+ and 9.x

**Problem:** MySQL 8.4 defaults to `caching_sha2_password`, which requires either SSL or RSA public key exchange. MygramDB disabled SSL for non-SSL connections but did not enable RSA key retrieval, causing `Authentication requires secure connection` errors.

**Solution:** Added `MYSQL_OPT_GET_SERVER_PUBLIC_KEY` option before `mysql_real_connect()`. This enables RSA public key retrieval for password exchange without SSL.

**File:** `src/mysql/connection.cpp`

### 2. Docker MySQL 9.x Compatibility

**Severity:** Medium — Docker containers fail to start with MySQL 9.x

**Problem:** Docker compose files used `--mysql-native-password=ON` (removed in MySQL 9.x) and `--binlog-format=ROW` (deprecated in MySQL 9.x, ROW is now the only format).

**Solution:** Removed both options from all docker-compose files. `MYSQL_OPT_GET_SERVER_PUBLIC_KEY` handles authentication, and ROW is the default binlog format.

**Files:**
- `docker-compose.yml`
- `docker-compose.bench.yml`
- `e2e/docker/docker-compose.yml`
- `support/testing/docker-compose.rpm-test.yml`

### 3. Flaky Truncate Memory Test

**Severity:** Low — Intermittent test failure

**Problem:** `test_truncate_releases_memory` timed out (30s) when replication backlog from prior tests delayed TRUNCATE event propagation.

**Solution:** Added `sync()` call before truncate and increased timeout to 60s.

**File:** `e2e/tests/memory/test_release.py`

## Migration Guide

### From v1.5.1

**No breaking changes.** Direct upgrade is safe.

1. Replace the binary or Docker image
2. If using Docker Compose, remove `--binlog-format=ROW` and `--mysql-native-password=ON` from MySQL command options (optional but recommended for MySQL 9.x readiness)
3. No configuration changes required

### MySQL 9.x Migration

To replicate from MySQL 9.x:

1. Upgrade MygramDB to v1.5.2+
2. Ensure `MYSQL_OPT_GET_SERVER_PUBLIC_KEY` is supported by your libmysqlclient (MySQL 8.0+ client library)
3. No MygramDB configuration changes needed — VECTOR columns are automatically handled
